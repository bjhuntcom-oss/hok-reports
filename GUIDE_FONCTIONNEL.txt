1. PRÉSENTATION GÉNÉRALE
================================================================================

HOK Reports est la plateforme de gestion documentaire interne du Cabinet HOK.
Elle permet aux avocats et collaborateurs de :

  • Enregistrer des sessions de consultation (audio)
  • Transcrire automatiquement les enregistrements (via Whisper AI)
  • Générer des rapports juridiques professionnels (via GPT-4o ou Claude)
  • Gérer les comptes rendus d'audience et les rôles hebdomadaires
  • Recevoir automatiquement les comptes rendus depuis WhatsApp
  • Analyser les messages WhatsApp avec l'IA (GPT-4o / Claude)
  • Suivre l'activité du cabinet via un tableau de bord

La plateforme est disponible en français et en anglais.

  Code source : https://github.com/bjhuntcom-oss/hok-reports
  Hébergement : Vercel (application) + Supabase (base de données PostgreSQL)

3. TABLEAU DE BORD
================================================================================

Le tableau de bord est la page d'accueil après connexion. Il affiche :

  • Nombre total de sessions enregistrées

  • Activité récente (dernières sessions et rapports)

C'est un résumé rapide de l'état du cabinet pour le jour.

4. SESSIONS D'ENREGISTREMENT
================================================================================

Une session représente un enregistrement audio d'une consultation ou réunion.

  CRÉER UNE SESSION :
  1. Cliquer sur "Nouvelle session" dans le menu
  2. Renseigner : titre, nom du client, email/téléphone (optionnel),
     référence dossier, description
  3. Enregistrer l'audio directement dans le navigateur (bouton micro)
  4. L'audio est automatiquement sauvegardé

  TRANSCRIRE UNE SESSION :
  1. Ouvrir une session existante
  2. Cliquer sur "Transcrire"
  3. L'IA (Whisper) convertit l'audio en texte
  4. La transcription apparaît avec segments horodatés

  GÉNÉRER UN RAPPORT :
  1. Après transcription, cliquer sur "Générer le rapport"
  2. Choisir le format : Synthèse / Standard / Détaillé
  3. L'IA (GPT-4o ou Claude) analyse et produit un rapport structuré
  4. Le rapport contient : résumé, points clés, actions, notes juridiques

  Les rapports sont rédigés dans un style juridique professionnel adapté
  au droit béninois et OHADA.


================================================================================
5. ENREGISTREMENT FLASH
================================================================================

L'enregistrement flash permet de capturer rapidement une note vocale
sans créer une session complète.

  1. Cliquer sur "Enregistrement flash" dans le menu
  2. Enregistrer le message vocal
  3. L'IA transcrit et extrait automatiquement :
     - Le nom du client mentionné
     - La référence du dossier
     - La catégorie (consultation, audience, réunion...)
     - Un titre et une description

  Idéal pour les notes rapides entre deux rendez-vous.


================================================================================
6. RAPPORTS
================================================================================

La section Rapports liste tous les rapports générés à partir des sessions.

  FONCTIONNALITÉS :
  • Consulter un rapport complet (résumé, points clés, actions, notes)
  • Filtrer par catégorie (consultation, audience, réunion, etc.)
  • Rechercher par nom de client ou référence
  • Exporter en PDF
  • Modifier le statut (brouillon → final → archivé)

  Chaque rapport est lié à une session d'origine et à un utilisateur.

7. RÔLES D'AUDIENCE ET INTÉGRATION WHATSAPP + IA
================================================================================

  QU'EST-CE QU'UN RÔLE D'AUDIENCE ?
  ──────────────────────────────────
  Un rôle d'audience est un tableau récapitulatif hebdomadaire qui fait
  le point des dossiers prévus pour être évoqués et des tâches attendues
  pour la semaine à venir.

  COMMENT ÇA FONCTIONNE ?
  ────────────────────────
  Le processus est entièrement automatisé :

  1. RÉCEPTION DES COMPTES RENDUS
     Au retour de chaque audience, l'avocat poste son compte rendu dans
     le groupe WhatsApp du cabinet — exactement comme il le fait déjà
     aujourd'hui. Il peut écrire librement, sans format imposé.

     L'application est connectée au groupe WhatsApp via l'API WhatsApp
     Cloud (Meta). Chaque message est automatiquement reçu et analysé.

  2. DÉTECTION INTELLIGENTE PAR IA
     L'IA (GPT-4o ou Claude) analyse chaque message WhatsApp entrant.
     Elle détecte automatiquement si le message parle d'une audience,
     d'un dossier, d'un renvoi, etc. — même si le message est écrit
     de manière informelle ou en langage courant.

     Pas besoin de format particulier. L'avocat écrit comme d'habitude.

     Le système utilise un score de confiance (0 à 1). Un message est
     considéré comme un compte rendu d'audience si le score dépasse 0.6.

     Si l'IA n'est pas disponible (pas de clé API), un système de
     détection par mots-clés prend le relais automatiquement.

  3. EXTRACTION AUTOMATIQUE DES DONNÉES
     Quand un compte rendu d'audience est détecté, l'IA extrait :
     • La date de l'audience
     • Le nom du client et de la partie adverse
     • La référence du dossier (RG, numéro...)
     • La juridiction et la chambre
     • Le nom de l'avocat
     • Un résumé professionnel et détaillé de l'audience
     • La date de la prochaine audience (si renvoi)
     • Les tâches à effectuer avant la prochaine audience
     • Des notes et observations juridiques

     Le résumé généré est professionnel et détaillé, rédigé dans un
     style juridique formel adapté au droit béninois et OHADA.
     L'IA déduit aussi les tâches actionnables à accomplir avant la
     prochaine audience (conclusions à rédiger, pièces à rassembler...).

     Deux moteurs IA sont supportés :
     • GPT-4o (OpenAI) — recommandé pour sa précision
     • Claude (Anthropic) — alternative performante

  4. GÉNÉRATION AUTOMATIQUE DU RÔLE HEBDOMADAIRE
     Le rôle de chaque semaine est calculé automatiquement en regroupant
     tous les comptes rendus dont la prochaine audience tombe dans cette
     semaine. Le tableau affiche :

     ┌──────────┬──────────────┬──────────────┬─────────┬───────────┬──────────┐
     │ Date     │ Dossier      │ Juridiction  │ Avocat  │ Résumé    │ Tâches   │
     ├──────────┼──────────────┼──────────────┼─────────┼───────────┼──────────┤
     │ lun. 24  │ Dupont c/    │ TPI Cotonou  │ Me      │ Le tribu- │ Préparer │
     │ févr.    │ SCI Immo     │ 1ère Ch.     │ Kossou  │ nal a...  │ concl... │
     └──────────┴──────────────┴──────────────┴─────────┴───────────┴──────────┘

     On peut naviguer de semaine en semaine avec les flèches ◀ ▶.

  5. SAISIE MANUELLE AUSSI POSSIBLE
     En complément de WhatsApp, un formulaire de saisie rapide est
     disponible directement sur la page Rôles pour entrer manuellement
     un compte rendu.

  IDENTIFICATION DES SOURCES
  ──────────────────────────
  Les comptes rendus reçus via WhatsApp sont identifiés par un badge
  vert "WhatsApp" dans la liste. Les saisies manuelles n'ont pas de badge.
  Le nom de l'expéditeur WhatsApp est aussi affiché.

8. ADMINISTRATION (Administrateur uniquement)
================================================================================

Accessible depuis le menu "Administration" (réservé aux administrateurs).

  GESTION DES UTILISATEURS :
  • Voir la liste de tous les utilisateurs inscrits
  • Approuver ou rejeter les demandes d'inscription (statut "en attente")
  • Bloquer / débloquer un utilisateur
  • Changer le rôle d'un utilisateur (utilisateur → administrateur)
  • Supprimer un compte utilisateur

  MOTEUR LLM :
  • Configurer la clé API OpenAI (pour transcription et rapports)
  • Vérifier le statut de configuration du moteur
  • Le moteur utilise Whisper pour la transcription et GPT-4o pour
    la génération de rapports et l'analyse des messages WhatsApp

  JOURNAL D'AUDIT :
  • Toutes les actions importantes sont tracées (connexions, créations,
    modifications, suppressions)
  • Consultable par l'administrateur

  STATISTIQUES :
  • Nombre de sessions, rapports, utilisateurs
  • Activité récente du cabinet

9. PARAMÈTRES
================================================================================

  LANGUE :
  • Français (par défaut) ou Anglais
  • Le changement est immédiat et sauvegardé localement

  INTÉGRATION WHATSAPP :
  La section WhatsApp dans les paramètres affiche :

  • Statut de connexion (Connecté / Non configuré)
  • URL du Webhook (à copier dans la console Meta)
  • Token de vérification (à copier dans la console Meta)
  • Statut de l'Access Token et du Phone Number ID
  • Format de message exemple pour les avocats
  • Simulateur de test (pour envoyer un faux message WhatsApp)

  POUR CONFIGURER WHATSAPP :
  1. Créer une application sur developers.facebook.com
  2. Activer l'API WhatsApp Business
  3. Dans la configuration du webhook de l'app Meta :
     - URL du webhook : [votre-domaine]/api/whatsapp/webhook
     - Token de vérification : hok-whatsapp-verify-2026
  4. Copier l'Access Token permanent et le Phone Number ID
  5. Les ajouter dans le fichier .env du serveur :
     WHATSAPP_ACCESS_TOKEN=votre_token
     WHATSAPP_PHONE_NUMBER_ID=votre_id
  6. S'abonner au champ webhook "messages"

  INFORMATIONS SYSTÈME :
  • Version de la plateforme
  • Éditeur (Cabinet HOK)
  • Moteur de traitement
