Section 01

Présentation & Stack technique

Vue d'ensemble de la plateforme et de ses technologies.

HOK Reports est une plateforme interne de gestion documentaire pour le Cabinet HOK (Avocats — Cotonou, Bénin). Elle automatise la transcription audio, la génération de rapports juridiques par IA, le suivi des audiences et la gestion des dossiers.

Mission — Automatiser la transcription, la synthèse et le suivi des audiences pour permettre aux avocats de se concentrer sur leur métier.

Section 06

Librairies internes (src/lib)

Détail de chaque module utilitaire du projet.

6.1 — llm.ts (Moteur IA Groq)

Fichier central de l'intelligence artificielle. Utilise exclusivement Groq (clé format gsk_...).

  • getGroqApiKey() — Récupère la clé (DB prioritaire, puis .env)
  • validateGroqKey(key) — Vérifie le préfixe gsk_
  • getGroqTranscribeClient() — Client OpenAI SDK pointé vers api.groq.com
  • transcribeAudio(buffer, lang) — Whisper whisper-large-v3-turbo → {text, segments}
  • extractSessionMetadata(text) — IA extrait titre, client, référence, catégorie, description
  • generateReport(text, client, ref, format, lang) — Génère rapport juridique complet via llama-3.3-70b-versatile
  • generateJSON(prompt, schema) — Génération JSON structuré avec validation

6.2 — sanitize.ts

  • sanitizeString(str) — Strip HTML, scripts, iframes, event handlers, entités dangereuses
  • validateEmail(email) — Regex standard
  • validatePassword(pwd) — 8+ car, majuscule, minuscule, chiffre, spécial, max 128
  • validateName(name) — 2-100 car, pas de caractères spéciaux dangereux
  • filterSessionFields(body) / filterReportFields(body) — Whitelist des champs modifiables

6.3 — audit.ts

  • logAudit({userId, action, entity?, entityId?, details?, ipAddress?, userAgent?}) — Enregistre dans AuditLog
  • getClientInfo(req) — Extrait IP (x-forwarded-for / x-real-ip) et User-Agent

6.4 — rate-limit.ts

  • rateLimit(key, maxRequests, windowMs) — Limiter en mémoire (Map). Fenêtre glissante. Retourne {allowed, remaining}
  • Nettoyage automatique des entrées expirées toutes les 5 minutes via setInterval

6.5 — whatsapp-parser.ts & whatsapp.ts

  • detectHearingReportAI(text) — IA détecte si un message contient un compte rendu (confiance ≥ 0.6)
  • parseWhatsAppMessageAI(text, sender?) — IA extrait données structurées + résumé professionnel juridique
  • isHearingReportKeywords(text) / parseWhatsAppMessageRegex(text) — Fallback regex
  • isWhatsAppConfigured() / getVerifyToken() / sendReportConfirmation() / sendParseErrorReply()

Section 08

Sécurité & Middleware

Toutes les mesures de protection implémentées.

Authentification

JWT via NextAuth v5Stratégie de session JWT signée avec AUTH_SECRET. Token contient id, role, picture.
Hachage bcrypt (salt 12)Mots de passe hachés avant stockage. Comparaison sécurisée via bcrypt.compare().
Google OAuthAuthentification complémentaire. Création auto de compte en pending. Linking d'email autorisé.

Autorisation

Vérification d'ownership (anti-IDOR)Chaque route API vérifie que l'utilisateur est propriétaire de la ressource. Les admins peuvent accéder à toutes les ressources.
Rôles différenciésAdmin : accès complet. User : uniquement ses propres données. Vérification double (middleware + route).
Approbation d'inscriptionNouveaux comptes en pending. Activation manuelle par admin. Comptes bloqués/rejetés ne peuvent pas se connecter.

Validation des entrées

Sanitisation XSSNettoyage systématique : HTML entities, scripts, iframes, event handlers (sanitizeString).
Validation stricteEmail regex, mot de passe (8+ car, maj, min, chiffre, spécial, max 128), nom (2-100 car).
Whitelist de champsSeuls les champs autorisés sont acceptés lors des PATCH (filterSessionFields, filterReportFields).

Section 09

Moteur IA (Groq) & Intégration WhatsApp

Architecture IA et pipeline WhatsApp.

9.1 — Moteur IA Groq

La plateforme utilise exclusivement Groq comme fournisseur IA. Aucune dépendance OpenAI ou Anthropic directe.

Transcription
Whisper

whisper-large-v3-turbo via Groq API

Génération
LLaMA 3.3

llama-3.3-70b-versatile pour rapports, métadonnées, parsing WhatsApp

La clé API Groq (gsk_...) peut être configurée via :

  • Variable d'environnement GROQ_API_KEY
  • Base de données (table SystemSetting, clé groq_api_key) — prioritaire
  • Panneau d'administration (sauvegarde en DB + runtime)

9.2 — Pipeline de génération de rapport

Le rapport généré par IA contient :

  • Titre — Généré automatiquement
  • Synthèse narrative — Résumé structuré en prose juridique
  • Points clés — Liste extraite des éléments importants
  • Actions à entreprendre — Tâches concrètes identifiées
  • Observations juridiques — Notes légales pertinentes
  • Suggestions IA — Typées : [LOI], [ARGUMENT], [DEFENSE], [OUVERTURE], [JURISPRUDENCE], [PREUVE]

9.3 — Intégration WhatsApp Cloud API

Le système est connecté via l'API WhatsApp Cloud (Meta). Variables d'environnement requises :

  • WHATSAPP_ACCESS_TOKEN — Token d'accès Meta
  • WHATSAPP_PHONE_NUMBER_ID — ID du numéro
  • WHATSAPP_VERIFY_TOKEN — Token de vérification webhook
  • WHATSAPP_BUSINESS_ACCOUNT_ID — ID du compte Business

9.4 — Pipeline WhatsApp

Lorsqu'un message WhatsApp est reçu :

  • Étape 1 — Détection IA : detectHearingReportAI(text) analyse si le message parle d'une audience (confiance ≥ 0.6)
  • Étape 2 — Extraction IA : parseWhatsAppMessageAI(text, sender) extrait date, client, référence, juridiction, chambre, avocat, adversaire, résumé professionnel, tâches
  • Étape 3 — Déduplication : vérification whatsappMessageId pour éviter les doublons
  • Étape 4 — Création HearingReport avec source: "whatsapp" et association à un utilisateur
  • Étape 5 — Confirmation envoyée via WhatsApp au sender

Si aucune clé IA n'est disponible, le système utilise le fallback regex (fonctionne uniquement pour les messages structurés).
